/search

/search/{searchentity} get

get

The Search API is optimized to find the specific entity you're looking for (e.g., a client, enrollments, etc.). It's designed to find the one result you're looking for (or maybe the few results you're looking for). Currently hmislynk supporting search on 'clients' entity only. Just like searching on Google, you sometimes want to see a few pages of search results so that you can find the item that best meets your needs. HMISLynk Search API provides up to 50 results for each search. However, the search results can be limited by the client. Note: Current supporting URI is /search/clients and /search/projects

get /search/{searchentity}

The Search API is optimized to find the specific entity you're looking for (e.g., a client, enrollments, etc.). It's designed to find the one result you're looking for (or maybe the few results you're looking for). Currently hmislynk supporting search on 'clients' entity only. Just like searching on Google, you sometimes want to see a few pages of search results so that you can find the item that best meets your needs. HMISLynk Search API provides up to 50 results for each search. However, the search results can be limited by the client. Note: Current supporting URI is /search/clients and /search/projects

Secured by oauth_2_0

The HMIS API supports OAuth 2.0 for authenticating all API requests.

  • Request
  • Response
  • Security

URI Parameters

  • searchentity: required (string)

Query Parameters

  • q: required (string)

    The search keywords for clients, as well as any qualifiers.Currently supported search fields are first name,middle name,last name,dob,source system id and ssn. For search on "date of birth" field the format MM-dd-YYYY should be used for the search keyword. The seach keywords for projects, as well as any qualifiers. Currenly supported search fields are projectName, projectCommonName, projectId.

  • sort: required (string)

    The sort field. Default- results are sorted by "created date".

    Example:

    created date
  • order: required (string - default: desc)

    The sort order if sort parameter is provided. One of asc or desc. Default- desc.

    Example:

    desc
  • startIndex: required (integer - default: 0)

    The start Index from where the search results should be returned. Default - 0

    Example:

    0
  • maxItems: required (integer - default: 50)

    the number or Items that should be included in the results . Default - 50

    Example:

    50
  • exclude: required (string)

    Excludes flieds from the search.Currently it suppoted value is "ssi" (to exclude source system id) only.

HTTP status code 200

OK – Everything is working.

HTTP status code 400

Bad Request – The request was invalid or cannot be served.

HTTP status code 401

Unauthorized – The request requires an user authentication.

HTTP status code 403

Forbidden – The server understood the request, but is refusing it or the access is not allowed.

HTTP status code 404

Not found – There is no resource behind the URI.

HTTP status code 500

Internal Server Error

Secured by oauth_2_0

Headers

  • Authorization: required (string)

    Used to send a valid OAuth 2 access token. Do not use an "access_token" query string parameter.

HTTP status code 401

Bad or expired token. This can happen if the user or the HMIS revoked or expired an access token. To fix, you should re- authenticate the user.

HTTP status code 403

Bad OAuth request (wrong consumer key, bad nonce, expired timestamp...). Unfortunately, re-authenticating the user won't help here.
